World Wide Web and HTML

Basics of the Web

The web is a network of computers communicating with each other. A user will access a webpage on their computer, by sending a message through the Internet, via HTTP protocol. That webpage is hosted on a server which is just another computer, and contains mostly HTML (Hypertext Markup Language) files. The server responds to the request by sending back the documents where it is interpretted by a web browser on the users computer.

HTML Basics

HTML files can contain four different things.

  • Text content - What you see
  • Markup - What it looks like
  • References to other documents - Images, Videos, etc
  • Links to other pages

Markup - Tags & Elements

HTML Markup is made up of Tags & Elements, which determines how your page looks. Each element generally contains an opening and closing tag, which surround content. Tags such as <img> (Images) and <br> (Break) do not contain a closing tag, and are called void tags. A good resource for all HTML tags and elements is this page.

Structure of an HTML Document

Page Structure

Part of writing good code, is making sure it has good structure. An HTML document should follow a tree like structure, with each element being a branch and indented one step further, to more easily identify that piece of code. Using a text editor such as Sublime or Atom, make coding and keeping good structure much easier.

All HTML elements are contained within a box and each indent in the page structure is one step further into the box.

Developer Tools

Using the developer tools in your web browser will allow you to see the elements and style of that web page, and are a good example of how an HTML document should be structured. Developers tools can also be very useful to test changes, before committing the changes to code.

Boxes

Everything on a web page is formatted in boxes, regardless of the shapes you see on the page. Each element is a box, which contains, content, padding, border and margin, which can each be customized to best suit the page. An element within an element, as also a box within a box.

Adding Style

CSS - Cascading Style Sheets

CSS allows you to give style to your document, and allows this to be done while avoiding repitition. CSS rules allows you to apply a class to each element, and apply the same style to all elements with that class. The rules will "cascade" down, until the most relevant rule for the element is found and can be applied. By allowing style to be easily applied to elements without repeating the same rules over and over, errors can be avoided.
